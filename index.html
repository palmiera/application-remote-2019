Pair programming

Two heads are better than one

Pair programming is a technique in wich two programmers work together side by side at one workstation.

One is writing the code on the computer and the other one is looking and reviewing each line of code. The one who's writing the code is called the Driver and the one that's reviewing each line of code that the Driver types in is called the Navigator.

The Navigator thinks about the big picture, strategy and what comes next while scanning for typographical error or bugs. That gives the Driver time to focus all the attention on the actual task.

The Pair programmers switch roles frequently about every five minutes or so.

"A problem shared, is a problem halved"

It's easier to catch mistakes when two people focus on the same code base.
